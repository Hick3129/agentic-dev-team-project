## 工作流程

- 從 `main` 建立分支 `test/initial-testing`
- 編寫 test cases（參考 `docs/requirements/acceptance-criteria.md`）
- 啟動前後端，執行測試案例
- 將結果填入 `test-execution-report.md`
- 缺血什么：
  - 所有 TC ✅ → sign-off 為 APPROVED
  - 有缺陷 → 放入 `bugs/` 並標記 NOT READY
- 提交並推送，建立 PR 到 `main`
- PR 內容使用 `docs/testing/sign-off.md` 作為說明
- 等待 `Validate Handoff`（檢查 `docs/testing/bugs/` 是否存在）
- merge 後流程結束
