# 前端手動測試指引 (Manual UI Testing Guide)

**目錄**：`docs/testing/manual-ui-testing-guide.md`  
**目標**：在瀏覽器中完整測試 Todo App 的 8 個功能案例 (TC-01–TC-08)  
**前提**：後端 API 已在 http://localhost:3000 運行，前端 static files 正在 serve（例如 http://localhost:8080）

---

## 準備步驟

1. **啟動後端**（如果尚未啟動）
   ```bash
   cd backend
   npm run dev   # 或: node dist/index.js
   ```
   確認 `curl http://localhost:3000/health` 回傳 `{"status":"ok"}`

2. **啟動前端靜態伺服器**
   ```bash
   cd frontend
   npx serve dist -l 8080
   ```
   打開瀏覽器訪問 http://localhost:8080，應看到「Todo App」標題與 empty list。

3. **打開瀏覽器 DevTools → Network**，確認 `/api/todos` 請求成功（200）。

---

## 測試案例執行

### TC-01: 新增待辦事項

**步驟**:
1. 在「標題」輸入框輸入 `買牛奶`
2. 可選：在「描述」輸入 `全脂`
3. 點擊「新增」按鈕

**預期結果**:
- 列表出現新項目，標題為「買牛奶」，描述為「全脂」
- 輸入框自動清空
- 網路請求：`POST /api/todos` 回傳 201 並包含 `id`

**✔ 通過 / ❌ 失敗**: _____

---

### TC-02: 空標題阻擋

**步驟**:
1. 不輸入任何文字，直接點擊「新增」按鈕

**預期結果**:
- 按鈕處於 disabled 狀態，或點擊後顯示錯誤訊息「標題必填」
- 不發出任何 API 請求（或返回 400）

**✔ 通過 / ❌ 失敗**: _____

---

### TC-03: 編輯待辦事項

**步驟**:
1. 先建立一個待辦（例如「寫報告」）
2. 點擊该项目右側的「編輯」按鈕
3. 修改標題為「寫月度報告」，可選修改描述
4. 點擊「儲存」

**預期結果**:
- 列表中的標題更新為「寫月度報告」
- 更新時間戳改变（可觀察 UI 上是否有顯示，或檢查 API 回傳 `updatedAt`）

**✔ 通過 / ❌ 失敗**: _____

---

### TC-04: 刪除待辦事項

**步驟**:
1. 在列表中任一待辦點擊「刪除」按鈕
2. 在確認對話框點擊「確定」

**預期結果**:
- 該項目從列表中消失
- 網路請求 `DELETE /api/todos/:id` 回傳 success

**✔ 通過 / ❌ 失敗**: _____

---

### TC-05: 切換完成狀態

**步驟**:
1. 在未完成的待辦旁點擊 checkbox
2. 觀察樣式變化（應出現刪除線）
3. 再次點擊 checkbox，恢復未完成狀態

**預期結果**:
- checkbox 狀態正確切換
- 列表重新渲染時樣式符合預期
- API 發送 `PATCH /api/todos/:id` 更新 `completed`

**✔ 通過 / ❌ 失敗**: _____

---

### TC-06: 篩選功能

**步驟**:
1. 建立至少 2 個待辦：一個未完成，一個已完成（透過切換 checkbox）
2. 點擊「未完成」篩選按鈕 → 只顯示未完成
3. 點擊「已完成」篩選按鈕 → 只顯示已完成
4. 點擊「全部」 → 顯示所有

**預期結果**:
- 篩選正確生效，列表只顯示符合條件的項目
- 切換時無錯誤

**✔ 通過 / ❌ 失敗**: _____

---

### TC-07: LocalStorage 持久化

**步驟**:
1. 新增幾個待辦（例如：A、B、C）
2. 關閉瀏覽器分頁（或刷新頁面）
3. 重新打開 http://localhost:8080

**預期結果**:
- 之前建立的待辦仍然存在（從 localStorage 恢復）
- 数量和内容一致

**✔ 通過 / ❌ 失敗**: _____

---

### TC-08: 響應式設計

**步驟**:
1. 打開瀏覽器 DevTools → 切換裝置工具栏（toggle device toolbar）
2. 選擇行動裝置尺寸（例如 iPhone 375x667）
3. 測試新增、編輯、刪除等操作，確認 UI 無破版、可按鈕都可點

**預期結果**:
- 所有元素在小螢幕下都可操作，无水平滾動條
- 表單輸入框、按鈕大小適當

**✔ 通過 / ❌ 失敗**: _____
