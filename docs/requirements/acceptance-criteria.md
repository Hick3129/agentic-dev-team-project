# 驗收標準 (Acceptance Criteria)

## AC-01: 新增待辦事项
- Given 我在首頁且表單可见
- When 我輸入標題「買牛奶」並點擊「新增」
- Then 列表中出現項目「買牛奶」，狀態為未完成
- And 標題輸入框自動清空

## AC-02: 不允許空標題
- Given 我在首頁
- When 我未輸入任何文字就點擊「新增」
- Then 按鈕保持禁用狀態，或顯示錯誤訊息「標題必填」

## AC-03: 編輯待辦事项
- Given 列表中有一個待辦「寫報告」
- When 我點擊該事项的「編輯」按鈕，修改標題為「寫月度報告」並儲存
- Then 列表中的標題更新為「寫月度報告」
- And 更新時間戳改变

## AC-04: 刪除待辦事項
- Given 列表中有一個待辦「逛街」
- When 我點擊「刪除」並在確認對話框點擊「確定」
- Then 該事项從列表消失
- And LocalStorage 中也不存在該項

## AC-05: 切換完成狀態
- Given 列表中有一個未完成的待辦「健身」
- When 我點擊該事项的 checkbox
- Then 该事項狀態改為已完成，並加上刪除線樣式
- And 再點一次則恢復為未完成

## AC-06: 篩選功能
- Given 列表中有以下事項：
  - 「晨跑」（未完成）
  - 「早餐」（已完成）
- When 我點擊「未完成」篩選按鈕
- Then 列表中只顯示「晨跑」
- When 我點擊「已完成」篩選按鈕
- Then 列表中只顯示「早餐」
- When 我點擊「全部」篩選按鈕
- Then 兩個事项都顯示

## AC-07: 資料持久化 (LocalStorage)
- Given 我新增了兩個待辦事項
- When 我關閉瀏覽器分頁後重新開啟
- Then 這兩個事项仍然存在，狀態不變

## AC-08: 響應式設計
- Given 我在手機裝置上開啟（view width < 768px）
- When 我操作新增、編輯、刪除等功能
- Then 所有按鈕和表單都可用，沒有溢出或破版
