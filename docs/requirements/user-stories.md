# 使用者故事 (User Stories)

## US-01: 新增待辦事項
**角色**：一般使用者  
**目標**：建立新的待辦事項  
**情境**：作為一個使用者，我想要快速新增一個待辦事項，以便記錄要做的事情。  

**驗收標準**：
- [ ] 表單有標題輸入框（必填）
- [ ] 表單有描述輸入框（選填）
- [ ] 點擊「新增」按鈕後，事項出現在列表中
- [ ] 標題為空時，按鈕應禁用或顯示錯誤

---

## US-02: 編輯待辦事項
**角色**：一般使用者  
**目標**：修改已新增的待辦事項內容  
**情境**：我想更新某個待辦的標題或描述。  

**驗收標準**：
- [ ] 每個事項旁有「編輯」按鈕
- [ ] 點擊後彈出表單，預載當前內容
- [ ] 儲存後列表立即更新

---

## US-03: 刪除待辦事項
**角色**：一般使用者  
**目標**：刪除不再需要的待辦事項  
**情境**：我不想看到舊的待辦，想把它們移除。  

**驗收標準**：
- [ ] 每個事項旁有「刪除」按鈕
- [ ] 點擊後需確認（confirm dialog）
- [ ] 確認後，該事項從列表消失

---

## US-04: 標記完成狀態
**角色**：一般使用者  
**目標**：快速標記事項為完成/未完成  
**情境**：我做完一件事，想把它標為完成。  

**驗收標準**：
- [ ] 每個事项有 checkbox
- [ ] 點擊 checkbox 切換狀態
- [ ] 完成的事项可以灰顯或加上刪除線

---

## US-05: 篩選待辦事項
**角色**：一般使用者  
**目標**：只顯示我關心的待辦（如未完成）  
**情境**：我只想看还没完成的事，不要被已完成的分心。  

**驗收標準**：
- [ ] 有三個篩選按鈕：全部 / 未完成 / 已完成
- [ ] 點擊後列表只顯示符合條件的事项
- [ ] 篩選狀態應持久化（localStorage）

---

## US-06: 資料持久化
**角色**：一般使用者  
**目標**：關閉瀏覽器後，我的待辦仍然存在  
**情境**：我花了時間整理待辦清單，不希望下次進來時全部消失。  

**驗收標準**：
- [ ] 使用 LocalStorage 儲存 Todo list
- [ ] 頁面載入時自動恢復_contents
- [ ] 若使用後端（SQLite），資料庫 schema 正確
